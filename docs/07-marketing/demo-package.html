4. 21 CFR Part 11 — E-Signatures & Audit Trail

Every approval, review, and sign-off in the system is captured as a 21 CFR Part 11 compliant electronic signature with SHA-256 content hash, server timestamp, signer identity, and IP address.

CapabilityHow it worksAPI
Sign a recordMeaning (AUTHORED/REVIEWED/APPROVED), signer identity, timestamp, SHA-256 hashPOST /api/signatures/sign
Verify integrityCompare current content hash vs hash at signing → detect tamperingPOST /api/signatures/verify
ALCOA+ audit trailWho, what, when, before/after, IP, user agent — immutable, sequentialGET /api/signatures/audit-trail/:type/:id
