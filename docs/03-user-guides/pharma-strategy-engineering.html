Hawkeye today — honest audit

Hawkeye is a production-grade mid-market pharma EQMS with strong domain modelling (150+ models) and solid workflow logic. The core audit-CAPA-deviation flow is fully built. The main risks are in cross-cutting platform primitives: no middleware-enforced multi-tenancy, no observability, no backup/PITR strategy, single-vendor LLM, no SSO/MFA.

Top-10 architecture gaps blocking 1000+ tenants + regulatory validation

  1. Multi-tenancy data isolation risk. No middleware-enforced scoping of queries. Every controller must manually filter by tenantId. One forgotten filter = cross-tenant breach. Fix: Mongoose plugin that auto-injects tenantId filter from req.tenantId.
  2. No distributed cache. In-memory Map cache limits horizontal scaling. Fix: Redis with tenant-keyed namespaces.
  3. Backup + PITR undefined. Hard deletes only · no soft-delete · no documented RTO/RPO. Fix: immutable delete log + daily Atlas snapshots + documented DR.
  4. Observability fully absent. No logs · metrics · traces. Fix: Pino + Prometheus + OpenTelemetry + Sentry.
  5. Hash-chaining incomplete. contentHashBefore/After exist but never computed. Fix: SHA-256 hashing on every CREATE/UPDATE in dataIntegrityLog.
  6. MFA feature-flag incomplete. No TOTP/SMS. Fix: WebAuthn + TOTP + SMS fallback. Enforce via tenant.requireMFA.
  7. No outgoing webhooks / async queue. Fix: BullMQ on Redis · standard signed-webhook retry queue.
  8. Single LLM vendor (OpenAI). Fix: LLM gateway with pluggable providers (Anthropic · Azure · local). Tenant-level choice for GxP paranoia.
  9. Admin scopes not enforced at query level. requireAdminScope() is route-only. Fix: query filter derived from req.user.adminScope.
  10. No IaC / multi-region. Fix: Terraform + cross-region MongoDB replica sets + regional object stores.

Multi-vertical extension — one platform, many industry packs

Each industry pack is a config + vocabulary overlay + module-specific schemas installed on the shared platform. No forking. The platform already supports this pattern via ModuleConfig.industryProfile + vocabularyOverrides.

Industry-pack anatomy

A vertical pack = (1) module on/off matrix · (2) regulatory control mapping (ISO clause → module) · (3) vocabulary overlay (audit/finding/capa per region) · (4) vertical-specific schemas (e.g. HACCP plans for food, APQP for auto) · (5) pre-built workflows + templates · (6) pre-validated IQ/OQ package (where applicable).

Tech debt inventory + validation burden

Immediate tech debt

  1. Two parallel CAPA implementations (capaModel + capaV2Models) — pick one; migrate + retire the other.
  2. Hardcoded state-machines alongside generic workflow engine — statusTrackerModel vs workflowSubjectModel. Consolidate onto the generic engine.
  3. No global search — Mongo text indexes only. Elastic/OpenSearch layer needed before 100+ tenants with 10M+ docs.
  4. Hard-delete everywhere — no soft-delete pattern. Required for audit/compliance. Retrofit deletedAt + audit-trail tombstones.
  5. ad-hoc console.log throughout — replace with structured logger (Pino).
  6. In-memory embedding cache at 4000-item cap — move to pgvector/Pinecone.
  7. Polling cron jobs — migrate to BullMQ async queue.
  8. No OpenAPI spec for v2 routes — swagger-jsdoc set up but not all routes annotated.
