Cross-cutting infrastructure

Electronic Signatures (21 CFR Part 11)

File: backend/src/routes/electronicSignatureRoutes.js

Endpoint: POST /api/electronic-signatures/sign

Generic e-signature endpoint accepts {recordType, recordId, signatureMeaning, authMethod}. Captures content SHA-256 hash, signer IP, user-agent. Signatures cannot be repudiated; logged to DataIntegrityLog. Used by document-control approvals, MRM completion, CAPA closure.

AI Audit Trail (FDA AI guidance · Jan 2025)

File: backend/src/services/ai/audit/aiAuditTrail.js

Endpoint: Internal: recordAiDecision() · POST /api/ai/decisions/outcome (user accept/reject)

Every AI decision logs feature, prompt-version, retrieval-set hashes, output, confidence, grounded flag, provider+model, latency, accept/reject outcome. Enables drift dashboard + regulator-ready evidence.

Universal Module Config

File: backend/src/middleware/universalPlatform.js

Endpoint: GET /api/universal-platform/config

Each tenant has modules.* boolean flags (EVENT_MANAGEMENT / DOCUMENT_CONTROL / RISK_MANAGEMENT / MANAGEMENT_REVIEW / CAPA / TRAINING / SUPPLIER_QUALITY / etc.). Frontend hides nav and 403s for disabled modules. Novex has all 15 enabled.

Role middleware

File: backend/src/middlewares/roleMiddleware.js

Endpoint: Used as middleware on every protected route

permit(...allowedRoles) gate. Roles normalised: tenantAdmin → tenant_admin, supplierUser → supplieruser, etc. Returns 403 with 'Forbidden: You don't have permission to access this resource.' if role not in allow-list.
