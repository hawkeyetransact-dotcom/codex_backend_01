1. Module overview

ModuleCAPA v2 — Corrective & Preventive Action
Pharma purposeFormalised corrective + preventive action lifecycle. Intake → triage → investigation → RCA → action plan → implementation → effectiveness review → closure. 5 mandatory approval gates.
Compliance basisISO 9001:2015 §10.2 · 21 CFR 211.192 · 21 CFR 820.100 (CAPA · medical device) · ICH Q10 §3.2.2 · EU GMP Chapter 1.4
Hawkeye routes/buyer/capas, /auditor/capas (frontend) · /api/capa-v2/* (backend)
Hawkeye model filebackend/src/models/capaV2Models.js (17 collections)
Tenant module flagmodules.CAPA

CAPA v2 supersedes the legacy single-collection capas with a 17-collection event-sourced model: every state transition writes a capa-v2-status-history row, every approval writes a capa-v2-approvals row, every action item is its own document, and every implementation artefact has its own capa-v2-implementation-evidence row. This makes it possible to satisfy 21 CFR Part 11 audit-trail requirements without separate logging — the data IS the audit trail. Every state transition is gated by the statusMachine.js state graph (no skipping states without admin override) and by role-based permits.

9. Known gaps + roadmap

  • Full UI for the 5 approval stages — Today the workspace shell is built but the per-tab forms are partially scaffolded. Need full UI for Investigation / RCA / Action Plan / Effectiveness drawers with required-field gating. HIGH
  • AI similarity / recurrence detector — capa-v2-similarity-links table exists but is empty. Plug embedding-based similarity (pgvector) to surface 'this CAPA looks like CAPA-2025-019 — recurrence?' suggestions at intake. MEDIUM
  • Action-item dependency graph — Action-items can have parents but the UI doesn't visualise the DAG. Add a Gantt-style or graph view. LOW
  • Effectiveness review reminder scheduler — When an action plan reviewWindow opens, send a notification to the reviewer (notificationOutbox). MEDIUM
  • Bulk-import legacy capas → v2 — Legacy capas collection is read-only. Provide a one-shot migration endpoint that maps old fields to v2 + preserves the audit trail. LOW
