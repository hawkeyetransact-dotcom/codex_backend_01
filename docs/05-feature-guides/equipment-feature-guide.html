1. Module overview

ModuleEquipment / Calibration
Pharma purposeTrack every GMP-relevant asset through its calibration lifecycle. Auto-escalate OVERDUE items. Failed calibrations trigger QUARANTINED status so the asset is blocked from production use.
Compliance basisISO 9001:2015 §7.1.5 (monitoring & measuring resources) · 21 CFR 211.68(b) (equipment qualification) · ICH Q7 §5.3
Hawkeye routes/asset-management (frontend) · /api/equipment (backend)
Hawkeye model filebackend/src/models/EquipmentModel.js
Tenant module flagmodules.ASSET_MANAGEMENT

Equipment moves ACTIVE ↔ UNDER_CALIBRATION ↔ INACTIVE / OUT_OF_SERVICE / QUARANTINED → RETIRED. Each calibration event captures performedBy + performedAt + result (PASS/CONDITIONAL/FAIL) + certificateRef + nextDueDays. FAIL → calibrationStatus=OVERDUE + status=QUARANTINED automatically.

9. Known gaps + roadmap

  • OVERDUE / DUE_SOON notification scheduler — Cron scans nextCalibrationDue; when within 30 days → DUE_SOON notification; when past → OVERDUE + email + Slack. HIGH
  • Auto-block from batch records when calibrationStatus != CURRENT — Batch creation should reject equipmentUsed[] entries pointing to QUARANTINED or OVERDUE assets. HIGH
  • Predictive calibration (Wave 3 ML) — P(failure) at next cal based on history + usage. MEDIUM
  • AI cert-extraction on calibration upload — Free Gemini Vision extracts values from cert PDF. LOW
