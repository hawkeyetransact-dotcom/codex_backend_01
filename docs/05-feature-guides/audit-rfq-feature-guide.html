1. Module overview

ModuleAudit Request / RFQ (marketplace audit flow)
Pharma purposeBuyer posts an audit RFQ. Multiple auditor orgs submit quotes. Buyer shortlists + awards. Awarded quote converts into a full audit request that enters the 8-phase audit lifecycle.
Compliance basisICH Q10 §2.7 · ISO 19011 §5.3 · contract audit governance
Hawkeye routes/rfqs, /rfqs/[id], /request-audit (frontend) · /api/rfqs, /api/audit-requests (backend)
Hawkeye model filebackend/src/models/auditRfqModel.js · auditRfqQuoteModel.js · auditRfqThreadModel.js
Tenant module flagmodules.RFQ

RFQs follow DRAFT → PUBLISHED → IN_QA → QUOTES_RECEIVED → SHORTLISTED → AWARDED → CONVERTED. Quote rows follow DRAFT → SUBMITTED → REVISED / WITHDRAWN / ACCEPTED / REJECTED. An awarded quote auto-creates an audit-requests-master row that enters the 8-phase audit lifecycle.

4.1 RFQ register

What it doesLists all RFQs with status + closingAt + invited-auditors count.
Menu / route/rfqs
Available to rolesbuyer · admin
Backend APIGET /api/rfqs

Click-by-click

  1. NAVIGATE Click 'RFQs' in the top nav

4.2 + New RFQ

What it doesBuyer drafts an RFQ.
Menu / route/rfqs · top-right '+ New RFQ'
Available to rolesbuyer
Backend APIPOST /api/rfqs

Click-by-click

  1. CLICK Click '+ New RFQ'
    → Form opens
  2. TYPE Fill title
    → required
  3. CLICK Pick supplierOrgId + siteId
    → required
  4. CLICK Pick productIds[]
  5. CLICK Pick closingAt (deadline for quotes)
    → required
  6. CLICK Click 'Save draft'
    → Status=DRAFT · rfqNumber=RFQ-XXXXXX

4.3 Publish

What it doesMove DRAFT → PUBLISHED so invited auditors can see it.
Menu / routeRFQ row · 'Publish' button
Available to rolesbuyer
Backend APIPOST /api/rfqs/:id/publish

Click-by-click

  1. CLICK Click 'Publish'
    → Status flips

4.4 Invite auditors

What it doesAdd specific auditor orgs to the invited list.
Menu / routeRFQ detail · 'Invite Auditors' button
Available to rolesbuyer
Backend APIPOST /api/rfqs/:id/invite

Click-by-click

  1. CLICK Click 'Invite Auditors'
    → Drawer with auditor org picker
  2. CLICK Pick auditorOrgIds[]
  3. CLICK Click 'Send invitations'
    → Each gets status=INVITED in invitedAuditors[]; notification sent

4.5 Submit quote (auditor)

What it doesAuditor submits a quote with line items + totals + proposed schedule.
Menu / routeRFQ detail (auditor view) · '+ Submit Quote' button
Available to rolesauditor
Backend APIPOST /api/rfqs/:id/quotes

Click-by-click

  1. CLICK Click '+ Submit Quote'
    → Form opens
  2. TYPE Fill lineItems[] (each: description + qty + unitPrice)
  3. TYPE Fill totals.subtotal + tax + total
  4. TYPE Fill proposedSchedule + assumptionsText
  5. CLICK Click 'Submit'
    → Quote row · status=SUBMITTED · RFQ status auto → QUOTES_RECEIVED

4.6 Shortlist + Award

What it doesBuyer narrows quotes, picks one to award.
Menu / routeRFQ detail · Quotes tab · 'Shortlist' / 'Award' buttons
Available to rolesbuyer
Backend APIPUT /api/rfqs/:id (status=SHORTLISTED) · POST /api/rfqs/:id/award (quoteId)

Click-by-click

  1. CLICK Click 'Shortlist' on the quotes you want to consider
    → Status=SHORTLISTED
  2. CLICK Click 'Award' on the winning quote
    → Status=AWARDED · awardedQuoteId set
  3. WAIT Auto-conversion
    → audit-requests-master row created with rfqId link · RFQ status=CONVERTED
