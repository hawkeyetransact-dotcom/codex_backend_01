1. Module overview

ModuleBatch / Manufacturing Records
Pharma purposeCapture the full manufacturing record for a single batch — BOM actuals, in-process tests, yield, equipment used, linked deviations — then drive QA review to final disposition (release / reject / quarantine).
Compliance basis21 CFR 211.188 (batch production records) · 21 CFR 211.192 (production record review) · EU GMP Annex 11 (electronic batch records) · ICH Q7 §6.5
Hawkeye routes/batch-records (frontend) · /api/batch-records (backend)
Hawkeye model filebackend/src/models/BatchRecordModel.js
Tenant module flagmodules.BATCH_RECORDS

Batches move MANUFACTURING → UNDER_REVIEW / PENDING_LAB_RESULTS / PENDING_QA_REVIEW → PENDING_DEVIATION_CLOSURE / PENDING_DISPOSITION → RELEASED / REJECTED / QUARANTINED. Operator captures BOM actuals + in-process tests + yields. QA reviews. VP disposes. Open deviations block release.

4. Feature catalogue (click-by-click)

4.1 Batch register

What it doesLists all batches with status chip + product + batchNumber + manufacturingDate.
Menu / route/batch-records
Available to rolesany tenant viewer
Backend APIGET /api/batch-records

Click-by-click

  1. NAVIGATE Click 'Batches' in the top nav
    → Page renders
  2. WAIT Spinner clears
    → Rows visible

4.2 + Create Batch dialog

What it doesOperator creates a new batch in MANUFACTURING state.
Menu / route/batch-records · top-right '+ Create Batch' button
Available to rolesoperator · admin
Backend APIPOST /api/batch-records

Click-by-click

  1. CLICK Click '+ Create Batch'
    → Dialog opens
  2. TYPE Fill batchNumber (e.g. NVX-2026-B015)
    → required
  3. TYPE Fill productName
    → required
  4. CLICK Pick manufacturingDate
    → required
  5. CLICK + Add billOfMaterials rows (materialName + theoreticalQty + actualQty + unit)
    → Multiple rows
  6. CLICK + Add inProcessTests rows (testName + spec + result + testedBy)
    → Multiple rows
  7. CLICK Click 'Save'
    → Row appears with status=MANUFACTURING

Fields captured

FieldRequiredValidation / valuesNotes
batchNumber Yes string · user-defined
productName Yes string
manufacturingDate Yes ISO date
billOfMaterials No [{materialName, theoreticalQty, actualQty, unit, supplierId, lot}]
inProcessTests No [{testName, spec, result, testedBy, testedAt, unit}]

4.3 Submit for review

What it doesOperator submits the batch for review. Routes to PENDING_LAB_RESULTS or PENDING_QA_REVIEW based on labResultsComplete flag.
Menu / routeBatch row · 'Submit for Review' button
Available to rolesoperator
Backend APIPOST /api/batch-records/:id/submit-for-review

Click-by-click

  1. CLICK Click 'Submit for Review'
    → Drawer with labResultsComplete + labResultsSummary + linkedDeviationIds
  2. CLICK Tick labResultsComplete=true (if QC done)
    → drives next state
  3. TYPE Fill labResultsSummary
  4. CLICK Click 'Submit'
    → Status flips to PENDING_QA_REVIEW (or PENDING_LAB_RESULTS if labResultsComplete=false)

4.4 QA review

What it doesQA reviews the batch record + verifies deviation closure + advances to disposition.
Menu / routeBatch row · 'QA Review' button (visible to QA role)
Available to rolesQA · admin
Backend APIPOST /api/batch-records/:id/qa-review

Click-by-click

  1. CLICK Click 'QA Review'
    → Drawer with qaReviewNotes + deviationsResolved + labResultsComplete checks
  2. TYPE Fill qaReviewNotes
    → required
  3. CLICK Tick deviationsResolved + labResultsComplete (both must be true)
    → if either false → status=PENDING_*
  4. CLICK Click 'Submit'
    → Status flips to PENDING_DISPOSITION (when both checks pass)

4.5 Dispose (VP)

What it doesFinal disposition. RELEASED auto-sets releaseDate.
Menu / routeBatch row · 'Dispose' button (visible to VP / tenant_admin)
Available to rolesVP · tenant_admin
Backend APIPOST /api/batch-records/:id/dispose

Click-by-click

  1. CLICK Click 'Dispose'
    → Drawer with decision picker
  2. CLICK Pick decision (RELEASED / REJECTED / REWORK / REPROCESS / QUARANTINED)
    → required
  3. TYPE Fill justification
    → required
  4. CLICK (Optional) e-sig
  5. CLICK Click 'Confirm + Sign'
    → Status flips; releaseDate auto-set if RELEASED

9. Known gaps + roadmap

  • Mandatory e-sig on RELEASE / REJECT — Wire /api/electronic-signatures/sign on /dispose. Captures meaning='RELEASED' or 'REJECTED'. HIGH
  • Stability sample auto-assignment on RELEASE — Auto-create retained sample record + scheduling per stability protocol. HIGH
  • AI yield anomaly detector — Wave 3 statistical detector flags yields outside ±2σ of last 30 batches. MEDIUM
  • Cross-link to deviation drawer — linkedDeviationIds clickable chips on batch detail. MEDIUM
