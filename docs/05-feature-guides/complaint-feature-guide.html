1. Module overview

ModuleComplaint Management
Pharma purposeCapture, investigate, link to CAPA, and close customer complaints. Determine MDR (Medical Device Report) reportability. Track regulatory reporting timelines.
Compliance basisISO 9001:2015 §10.2 · 21 CFR 211.198 (drug complaint files) · 21 CFR 820.198 (medical device complaint files) · EU GVP
Hawkeye routes/complaint-manager (frontend) · /api/complaints (backend)
Hawkeye model filebackend/src/models/ComplaintModel.js
Tenant module flagmodules.COMPLAINT_MANAGEMENT

Complaints follow OPEN → UNDER_INVESTIGATION → PENDING_CAPA → CAPA_IN_PROGRESS → PENDING_CLOSURE → CLOSED (or CANCELLED). isMedicalDeviceReport flag triggers the MDR reportability assessment; requiresRegulatoryReporting gates the regulatory submission timeline. linkedCAPAIds bridges to the CAPA-v2 workspace.

9. Known gaps + roadmap

  • AI complaint triage agent — Pre-suggest severity, MDR flag, similar past CAPAs. HIGH
  • MDR / FAR deadline countdown UI — Visible counter on the row when requiresRegulatoryReporting=true. Email reminder at 80% elapsed. HIGH
  • FDA MedWatch eMDR submission integration — Programmatic 3500A submission API. MEDIUM
  • Customer notification template — Closure notes auto-formatted into a customer-facing letter. LOW
