1. Module overview

ModuleRisk Register (FMEA)
Pharma purposeFMEA-style hazard register. Capture process-step risks (failure mode → effect → cause), score with Severity × Occurrence × Detectability = RPN, add mitigations until residual risk acceptable.
Compliance basisISO 9001:2015 §6.1 (risk-based thinking) · ICH Q9 (R1) Quality Risk Management · ISO 14971 (med device risk mgmt)
Hawkeye routes/risk-register (frontend) · /api/risk-items (backend)
Hawkeye model filebackend/src/models/RiskItemModel.js
Tenant module flagmodules.RISK_MANAGEMENT

Each risk row captures S/O/D scores (1-10) which auto-compute RPN = S × O × D. RPN bands: <60 LOW, 60-124 MEDIUM, 125-199 HIGH, ≥200 CRITICAL. Risks move OPEN → MITIGATED (after at least one mitigation completed and residual S/O/D entered) → ACCEPTED → CLOSED. An optional Wave-2 AI brainstormer pre-suggests failure modes / effects / causes from a process description.

4.3 + Log Risk dialog

What it doesCapture a new FMEA risk row. RPN computes automatically.
Menu / route/risk-register · top-right '+ Log Risk' button
Available to rolesany tenant user
Backend APIPOST /api/risk-items

Click-by-click

  1. CLICK Click '+ Log Risk'
    → Dialog opens with FMEA form
  2. TYPE Fill processStep (e.g. 'Blending · Line 2')
    → required
  3. TYPE Fill failureMode (one sentence)
    → required
  4. TYPE Fill failureEffect (consequence)
    → required
  5. TYPE (Optional) Fill failureCause
  6. CLICK Pick severity (1-10)
    → required; 10 = catastrophic patient harm
  7. CLICK Pick occurrence (1-10)
    → required; 10 = highly likely
  8. CLICK Pick detectability (1-10)
    → required; 10 = nearly impossible to detect
  9. CLICK Pick riskCategory (QUALITY / SAFETY / REGULATORY / OPERATIONAL / ENVIRONMENTAL / FINANCIAL / CUSTOM)
    → default QUALITY
  10. CLICK Click 'Save'
    → Row appears with RPN auto-computed; band auto-set

9. Known gaps + roadmap

  • Wire AI Brainstormer button into the +Log Risk dialog — Today the API works but no UI button. Add a 'Suggest failure modes with AI' button next to processStep input. HIGH
  • Mandatory e-sig on ACCEPTED + CLOSED — Wire /api/electronic-signatures/sign for residual-risk acceptance. HIGH
  • nextReviewDate scheduler — Field + cron + reminder for periodic re-evaluation of MITIGATED risks. MEDIUM
  • Heatmap view (Severity × Occurrence) — Visual grid where each cell is a risk. Drill-down on click. MEDIUM
  • Cross-link to source records (deviation/CAPA/audit) — Today sourceType + sourceRefId are captured; surface them as clickable chips. LOW
